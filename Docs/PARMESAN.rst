PARMESAN
========

PARMESAN (PARametric Mapping using Estimation of Steady-state And traNsient signals) is a framework for quantitative parameter mapping from multi-pulse acquisition sequences. It includes tools for both steady-state and transient signal analysis, as well as an RF pulse simulation utility.

The following commands are available:

* `qi transient`_
* `qi ss`_
* `qi rf_sim`_

qi transient
------------

Fits a transient-type PARMESAN model to MUPA (Multi-Pulse Acquisition) data. Can fit either a standard B1 model or an MT (Magnetization Transfer) model.

**Example Command Line**

.. code-block:: bash

    qi transient mupa_data.nii.gz < input.json

**Example JSON File**

.. code-block:: json

    {
        "MUPA" : {
            ...
        }
    }

The JSON file should contain the MUPA sequence parameters.

*Important Options*

* ``--mt``

    Use the MT model instead of the default B1 model.

* ``--T2b``

    T2 of the bound pool (default 12 µs). Only used with the MT model.

* ``--lineshape``

    Path to a lineshape JSON file (generated by ``qi lineshape``). Only used with the MT model.

qi ss
-----

Fits a steady-state PARMESAN model to MUPA data. Supports T1-only, T1/T2, and MT models.

**Example Command Line**

.. code-block:: bash

    qi ss mupa_data.nii.gz < input.json

*Important Options*

* ``--T2``

    Fit the T2 model (T1 and T2).

* ``--MT``

    Fit the MT model (requires a lineshape file).

* ``--lineshape``

    Path to a lineshape JSON file (generated by ``qi lineshape``). Required when ``--MT`` is specified.

qi rf_sim
---------

Calculates RF pulse properties for use with PARMESAN models. Takes a JSON file specifying RF pulse shapes and calculates effective flip angles and integrals needed for the PARMESAN signal models.

**Example Command Line**

.. code-block:: bash

    qi rf_sim pulses.json

The input JSON file should contain pulse definitions with B1x, B1y waveform components and timestep values. The output (pulse properties) is printed to ``stdout`` as JSON.

*Important Options*

* ``--uT, -u``

    Input pulse amplitudes are in microTesla instead of radians per second.
